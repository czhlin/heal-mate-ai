# 未来方向：我们可以一起做些什么？

HealMate AI 目前只完成了一个 MVP（最小可行性产品），它的潜力远不止于此。
如果你想参与开源，但不知道该做什么，这里有一些我们未来非常想实现的方向（也就是你可以认领的 Task！）。

## 📈 1. 数据的“升维”（多模态接入）
目前用户必须手打文字告诉 AI 自己的身体状况，这太累了。
- **体检报告解析**：允许用户上传 PDF/图片的体检报告，调用 OCR 或者大模型的视觉能力，自动提取血糖、血脂等异常指标，并转化为 AI 健康计划的限制条件。
- **可穿戴设备接入**：能不能接入 Apple Health、小米健康或者智能手表的 API？每天自动同步用户的步数和睡眠时长，而不是让用户手动打卡。

## 🎨 2. 交互的“降维”（更轻松的使用体验）
- **语音打卡**：接入浏览器的麦克风功能，用户只需要说一句“我今天加了个班，吃了顿麦当劳”，AI 自动提取情绪和饮食记录，完成打卡并给出安慰。
- **微信/企微/飞书机器人**：虽然 Streamlit 网页很好，但健康打卡这种高频操作，如果在微信里直接和机器人聊天就能完成，用户的留存率会大幅提升。我们需要把核心的 `services` 暴露成 API，然后接入机器人。

## 🧠 3. AI 的“进化”（记忆与专业性）
- **长程记忆管理 (RAG)**：现在的 AI 只记得你最初填的档案。如果能引入向量数据库（Vector DB），让 AI 记住你上个月生过什么病、你最讨厌吃什么，它就会变得越来越像一个懂你的老朋友。
- **专业知识库挂载**：引入权威的医学指南和营养学文档，防止大模型产生“幻觉”胡乱给出医疗建议（AI 不应替代医生，但可以提供符合指南的营养建议）。

## 📊 4. 数据可视化（成就感体系）
- **用户成长看板**：在前端 `pages/` 增加一个数据面板页面，用折线图展示用户最近一个月的睡眠趋势，用热力图（类似 GitHub 的绿格子）展示打卡记录。
- **周报与月报**：AI 每周自动总结用户这一周的表现，并生成一段有温度的“每周健康寄语”。
