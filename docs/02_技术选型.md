# 技术选型：为什么我们这么搭积木？

对于初学者来说，看到一堆新技术名词往往会感到迷茫。这里为你解密 HealMate AI 的技术拼图，以及我们“为什么”这样选。

## 1. 语言：Python
- **为什么选它**：Python 是 AI 时代的绝对主流语言。无论是调用大模型 API，还是处理数据，Python 的生态都无与伦比。它语法简洁，对初学者极其友好。

## 2. 前端框架：Streamlit
- **什么是它**：Streamlit 是一个允许你**只写 Python 代码就能生成现代 Web 网页**的框架。
- **为什么选它**：
  - **极速开发**：不用写 HTML、CSS、JavaScript。一句 `st.button("点击")` 就能在网页上画出一个按钮。
  - **状态管理友好**：自带 `st.session_state`，非常适合做对话、表单等状态保留。
  - **专注业务**：让我们把有限的精力放在 AI 逻辑和用户体验上，而不是纠结前端元素怎么居中。

## 3. 大模型 AI：DeepSeek API + OpenAI SDK
- **什么是它**：DeepSeek 是一家国产优秀的 AI 公司，我们使用了他们的模型作为底层“大脑”。由于他们的 API 格式完全兼容 OpenAI，所以我们代码里用的是 `openai` 这个官方 Python 包。
- **为什么选它**：
  - 智商极高，尤其是中文语境下的共情能力和逻辑推理能力。
  - 相比国外模型，网络调用更稳定，成本极低，非常适合个人开发者和开源项目起步。

## 4. 数据库：SQLite 3
- **什么是它**：Python 内置的一个轻量级关系型数据库，所有数据都存在本地的一个 `.sqlite` 文件里。
- **为什么选它**：
  - **零配置**：不需要安装 MySQL、PostgreSQL 那些庞大的数据库软件，开箱即用。
  - **容易理解**：用最标准的 SQL 语句就能增删改查。
  - **演进路线**：MVP（最小可行性产品）阶段用 SQLite 跑通逻辑；未来如果用户量大了，代码结构不变，只需换个连接字符串就能平滑迁移到云端数据库（如 Supabase/PostgreSQL）。

## 5. 工程化工具：Pytest + Ruff
- **Pytest**：用于写自动化测试。修改代码后，跑一下测试就知道有没有把别的地方弄坏。
- **Ruff**：一个极速的 Python 代码检查和格式化工具（Linter & Formatter）。它保证了开源项目里所有人提交的代码风格都是统一的、整洁的。
