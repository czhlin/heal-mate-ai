# 前置学习路线：从小白到能改代码需要学什么？

不要被代码仓库里密密麻麻的文件吓到。在这个 AI 辅助编程的时代，你不需要把厚厚的教科书从头背到尾。
这里为你提供了一条**“最少必要知识”**学习路线。

## 🎯 阶段 1：Python 基础（1-2天）
你只需要知道最核心的概念，剩下的都可以让 AI 帮你写。
- **变量与数据类型**：什么是字符串 (`str`)、整数 (`int`)、列表 (`list`)、字典 (`dict`)？
  - *重点看：字典 (dict) 怎么读取和修改，因为我们的数据大多是这种格式。*
- **控制流**：`if / else` 判断，`for` 循环。
- **函数**：`def` 是什么？参数怎么传？`return` 是什么意思？
- **如何学习**：B站搜“Python 基础 1小时入门”，或者直接问 AI（如：“用大白话给我讲讲 Python 的字典是什么”）。

## 🎯 阶段 2：Git 和 GitHub（半天）
作为开源协作的基础，你需要知道怎么把代码推上来。
- **核心命令**：
  - `git clone`（下载代码到本地）
  - `git checkout -b 分支名`（创建自己的工作区）
  - `git add .` 和 `git commit -m "说明"`（保存你的修改）
  - `git push`（把修改传到云端）
- **PR (Pull Request)**：怎么在 GitHub 上发起代码合并请求。
- **如何学习**：可以直接在 Cursor/Trae 等编辑器里使用可视化界面点点点，不需要死记命令。

## 🎯 阶段 3：Streamlit 极速入门（1天）
这是我们用来画网页的库，非常简单好玩。
- **怎么显示文字**：`st.write()`, `st.markdown()`
- **怎么弄输入框**：`st.text_input()`, `st.button()`
- **核心难点（重中之重）**：`st.session_state`（会话状态）。
  - *理解它：因为网页是无状态的，你点一下按钮，代码会从头重新跑一遍。如果你想记住刚才输入了什么，就必须把它存在 `st.session_state` 里。*
- **如何学习**：看 [Streamlit 官方文档](https://docs.streamlit.io/) 的 Get Started 部分。

## 🎯 阶段 4：SQL 与数据库基础（1天）
我们用了 SQLite，你需要懂一点点数据库的黑话。
- **建表 (CREATE TABLE)**：相当于在 Excel 里新建一个 Sheet，并定义好表头。
- **增 (INSERT)**：插入一条新数据。
- **查 (SELECT)**：比如 `SELECT * FROM users WHERE id = '123'`。
- **改 (UPDATE)** 和 **删 (DELETE)**。
- **如何学习**：你完全可以让 AI 帮你写 SQL，你只需要能看懂它写的句子大概是什么意思就行。

## 🤖 终极作弊码：如何用好 AI 辅助开发
在本项目中，你不需要手写所有代码。我们强烈推荐你使用 Cursor 或 Trae 这样的 AI 编程工具。

**不好的提问：**
> “帮我加个登录页面” （太宽泛，AI 会乱写一通破坏架构）

**好的提问（懂架构后的提问）：**
> “我想在 `src/pages/home.py` 里加一个‘忘记密码’的按钮。如果点击了，请调用 `src/services/auth_service.py` 里的一个新方法 `reset_password`（这个方法你需要帮我在 service 和 repo 层实现一下，用 SQLite 存）。最后记得在 `tests/` 里加上对应的 pytest 测试用例。”

只要你能用人话把**“在哪一层，调用什么，存去哪里”**描述清楚，AI 就能写出完美符合规范的代码！
